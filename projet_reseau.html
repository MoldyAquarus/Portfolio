🌐 Les fondations de l'Administration Réseau

📌 Contexte et Description

En complément de l'administration système, les cours de réseaux du BUT Informatique abordent la manière dont les équipements informatiques communiquent entre eux. Bien que cette matière représente un véritable défi technique pour moi, elle m'a permis d'acquérir le vocabulaire et les concepts essentiels pour comprendre l'infrastructure sur laquelle reposent les applications que je développe.

🎯 Objectifs techniques

  • Comprendre le modèle OSI et le modèle TCP/IP.
  • Maîtriser l'adressage IP (IPv4) et le calcul de sous-réseaux (masques).
  • Configurer la communication de base entre plusieurs machines.

💡 Ma démarche et mes preuves d'apprentissage (Compétence Administrer)

Plutôt que de viser une expertise en architecture réseau complexe, ma démarche a été de sécuriser les connaissances fondamentales nécessaires à un développeur :

  • Analyse de trames : Lors des travaux pratiques, j'ai utilisé des outils de capture réseau (comme Wireshark ou Packet Tracer) pour observer concrètement comment les données transitent sur un réseau.
  • Configuration IP : J'ai appris à configurer des adresses IP statiques et dynamiques (DHCP), et à calculer des plages de sous-réseaux pour segmenter une infrastructure.
  • Diagnostic : J'ai acquis les réflexes de base pour diagnostiquer une panne réseau avec les commandes système (`ping`, `ipconfig/ifconfig`, `traceroute`).

🎓 Ce que cette preuve démontre

Cette preuve témoigne de ma persévérance face à la difficulté. Le réseau n'étant pas mon domaine de prédilection, j'ai dû fournir un travail supplémentaire pour valider ces acquis. Aujourd'hui, je suis capable de dialoguer efficacement avec un administrateur système ou réseau, car je comprends les contraintes d'adressage et de routage auxquelles mes applications logicielles seront soumises lors de leur déploiement.

🛠️ Notions abordées

Protocoles : TCP/UDP, IP, DHCP, DNS.
Outils : Commandes de diagnostic réseau, Packet Tracer.
