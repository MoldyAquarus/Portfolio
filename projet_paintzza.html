🍕 Projet Paintzza (JavaFX)

📌 Contexte et Description

Réalisé au cours de ma formation en BUT Informatique, le projet Paintzza est une application "Desktop" (bureau) de prise de commande de pizzas. Réalisé [seul / en groupe de X personnes], ce projet avait pour but de nous initier à la création d'interfaces graphiques interactives en Java.

🎯 Objectifs techniques

  • Créer une Interface Homme-Machine (IHM) fluide et ergonomique.
  • Structurer le code en respectant scrupuleusement le patron de conception MVC (Modèle-Vue-Contrôleur).
  • Gérer les événements utilisateurs (clics, sélections d'ingrédients, validation du panier).

💡 Ma démarche et mes preuves d'apprentissage (Compétence Réaliser)

Ce projet illustre parfaitement ma capacité à élaborer une application orientée objet avec une interface visuelle complexe :

  • Conception de l'Interface (Vue) : J'ai utilisé l'outil Scene Builder pour concevoir les fichiers FXML. Cela m'a permis de séparer proprement le design de l'application (la Vue) de sa logique (le Contrôleur), facilitant ainsi la maintenance du code.
  • Logique Métier (MVC) : J'ai programmé en Java les contrôleurs permettant de lier les actions de l'utilisateur (ex: ajouter une option à la pizza) à la mise à jour des données en temps réel (ex: calcul du prix total dynamique).
  • Tests et Débogage : L'un des défis majeurs a été de gérer les états de l'interface pour éviter qu'un utilisateur valide une commande vide".

Ce que ce projet démontre

Le projet Paintzza démontre que je suis capable de dépasser le stade de la simple "ligne de commande" (CLI) pour fournir une application dotée d'une véritable interface utilisateur graphique (GUI). Il valide ma compréhension de l'architecture logicielle (séparation des responsabilités via le MVC) et ma maîtrise de la programmation orientée objet en Java.

🛠️ Technologies et outils utilisés

Langage : Java
Framework & Outils : JavaFX, Scene Builder, FXML, [Eclipse / VS Code]
