🧱 SAÉ 3 & 4 : Site E-Commerce "Tableau LEGO"

📌 Contexte et Description

Ce projet constitue la SAÉ (Situation d'Apprentissage et d'Évaluation) centrale de ma deuxième année de BUT. L'objectif global est de concevoir un site e-commerce complet permettant à un client de téléverser une image personnelle afin de la transformer en un tableau physique composé de briques LEGO.

La particularité de ce projet réside dans son architecture modulaire : chaque fonctionnalité (front-end web, traitement d'image, algorithme d'optimisation, base de données) est développée individuellement dans des langages différents, avant d'être assemblée lors d'un travail en équipe.

🎯 Objectifs techniques du cahier des charges

  • Front-End (PHP Objet) : Création de l'interface utilisateur, upload d'images, gestion des comptes clients, devis PDF et paiements.
  • Analyse et Traitement (Java) : Pixellisation de l'image cliente et association avec la palette de couleurs des briques LEGO disponibles via la base de données.
  • Algorithmique (C) : Développement d'un algorithme de pavage optimisé pour remplacer les pixels par des briques de tailles variées (1x1 à 2x4) en minimisant le coût.
  • Base de données (PostgreSQL) : Conception d'un modèle relationnel sécurisé pour gérer les utilisateurs, les commandes, les factures et le catalogue de briques.

💡 Ma démarche et mes preuves d'apprentissage (Compétence Réaliser)

À travers ce projet ambitieux, j'ai pu démontrer concrètement ma maîtrise de la compétence Réaliser :

  • Élaborer une application complexe : J'ai dû coder des algorithmes spécifiques (en C et Java) pour traiter les images, tout en assurant le développement d'un serveur Web robuste en PHP. Cela m'a appris à choisir les bonnes structures de données selon le langage.
  • Faire communiquer des modules hétérogènes : L'un des plus grands défis (et réussites) a été de faire dialoguer le Front-End (PHP) avec les composants Back-End (Java/C) afin d'assurer un flux de données fluide et sécurisé de l'upload de l'image jusqu'à la proposition de devis.
  • Maintenir et sécuriser : Sensibilisé aux enjeux professionnels, j'ai mis en place un code propre et commenté (en anglais, comme exigé par le cahier des charges) ainsi que des requêtes SQL sécurisées pour prévenir les failles.

🚧 Regard critique et difficultés

La difficulté majeure a été de faire communiquer les différents modules de ce projet entre eux, chose que je n'avais fais avant qu'avec des bases de données. Le Java a été particulièrement coriace avec l'utilisation d'une API.

🛠️ Technologies et outils utilisés

Langages : PHP (Objet), Java, C, SQL, HTML/CSS
Outils : Git/GitHub, Environnement Linux
